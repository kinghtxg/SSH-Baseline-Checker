# SSH-Baseline-Checker 测试计划

## 1. 测试目标

本测试计划旨在验证 SSH-Baseline-Checker v0.1 版本的功能完整性、稳定性、安全性和性能，确保其满足腾讯测试规范要求，为生产环境部署提供质量保障。

## 2. 测试范围

| 模块 | 测试内容 | 优先级 |
|------|----------|--------|
| 核心引擎 | SSH连接、命令执行、交互式Shell、Root提权 | P0 |
| 数据库模型 | 所有模型CRUD、关联查询、数据完整性 | P0 |
| Web API | 所有RESTful端点、参数验证、错误处理 | P0 |
| 前端页面 | 页面渲染、交互功能、数据绑定 | P1 |
| 工具模块 | Ping/Telnet/Curl/正则测试 | P1 |
| 截图引擎 | ANSI清理、换行计算、图片生成 | P2 |
| 安全特性 | 认证授权、敏感信息保护、输入过滤 | P0 |
| 性能 | 并发连接、大数据量处理、响应时间 | P2 |
| 兼容性 | 不同SSH服务器、Python版本、操作系统 | P1 |

## 3. 测试策略

### 3.1 单元测试
- 使用 `pytest` 框架
- 对每个模块独立测试
- Mock外部依赖（SSH连接、数据库等）
- 代码覆盖率目标：≥80%

### 3.2 集成测试
- API端点集成测试
- 数据库操作集成
- 前端-后端数据流验证

### 3.3 系统测试
- 完整业务流程验证
- 多用户并发场景
- 异常场景恢复

### 3.4 安全测试
- 输入验证测试
- 认证授权测试
- 敏感信息泄露测试
- SQL注入防护

### 3.5 性能测试
- 并发连接压力测试
- 大数据量报告生成测试
- 响应时间基准测试

## 4. 测试环境要求

### 4.1 硬件环境
- CPU: 4核以上
- 内存: 8GB以上
- 存储: 50GB可用空间
- 网络: 可访问测试SSH服务器

### 4.2 软件环境
- 操作系统: Windows 10/11, Ubuntu 20.04+
- Python: 3.8-3.11
- 数据库: SQLite (默认), MySQL (可选)
- SSH测试服务器: OpenSSH 7.0+

### 4.3 测试工具
- pytest 8.2.0
- pytest-cov 5.0.0
- pytest-flask 1.3.0
- paramiko 3.4.0 (Mock)
- Pillow 10.3.0

### 10.1 质量要求
1. 所有P0、P1优先级测试用例100%通过

2. 页面加载时间 ≤3秒
3. 内存使用符合预期
4. 无内存泄漏

### 10.3 安全要求
1. 通过安全扫描
2. 无高危安全漏洞
3. 敏感信息保护符合要求

## 11. 缺陷管理规范

### 11.1 缺陷等级定义
| 等级 | 描述 | 修复优先级 |
|------|------|------------|
| Blocker | 系统崩溃、数据丢失、主要功能完全失效 | 立即修复 |
| Critical | 主要功能部分失效、安全漏洞 | 高优先级 |
| Major | 次要功能失效、影响用户体验 | 中等优先级 |
| Minor | 界面问题、提示信息错误 | 低优先级 |
| Trivial | 拼写错误、格式问题 | 酌情修复 |

### 11.2 缺陷处理流程
1. **发现与报告**：测试人员发现缺陷，记录详细信息
2. **确认与分配**：开发负责人确认缺陷，分配给开发人员
3. **修复与验证**：开发人员修复缺陷，测试人员验证
4. **关闭与归档**：缺陷修复通过后关闭，记录修复信息
